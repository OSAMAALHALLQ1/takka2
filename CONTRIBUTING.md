# 🤝 دليل المساهمة

شكراً لاهتمامك بالمساهمة في **تكة**! هذا الدليل يساعدك في البدء.

---

## 🚀 كيفية البدء

### 1. استنساخ المشروع
```bash
git clone https://github.com/OSAMAALHALLQ1/takka.git
cd takka
```

### 2. إنشاء فرع جديد
```bash
git checkout -b feature/اسم-الميزة
# أو للإصلاحات:
git checkout -b fix/اسم-الإصلاح
```

### 3. تثبيت المكتبات
```bash
npm install
```

### 4. بدء التطوير
```bash
npm run dev
```

---

## 📝 معايير الكود

### التسمية
- استخدم CamelCase للدوال والمتغيرات: `handleSubmit`, `userData`
- استخدم PascalCase للـ Components: `LoginComponent`, `WaiterView`
- استخدم kebab-case للملفات: `login-component.jsx`

### التعليقات
- استخدم العربية للتعليقات المهمة
- اشرح **لماذا** وليس **ماذا**
```jsx
// ✅ جيد
// نتحقق من وجود البيانات قبل العرض لتجنب الأخطاء
if (data) { ... }

// ❌ سيء
// تحقق من البيانات
if (data) { ... }
```

### التنسيق
```bash
npm run lint
```

## 🐛 الإبلاغ عن الأخطاء

عند فتح issue، أرجو تقديم:

1. **الوصف الواضح** للمشكلة
2. **خطوات إعادة الإنتاج**
3. **السلوك المتوقع**
4. **السلوك الفعلي**
5. **لقطات الشاشة** (إن أمكن)

```markdown
## 🐛 الخطأ
وصف المشكلة هنا

## 📋 خطوات إعادة الإنتاج
1. اذهب إلى ...
2. اضغط على ...
3. لاحظ الخطأ

## ✅ السلوك المتوقع
ما يجب أن يحدث

## ❌ السلوك الفعلي
ما الذي يحدث بالفعل

## 📸 لقطات الشاشة
[إذا أمكن]
```

---

## 💡 اقتراح ميزات جديدة

عند اقتراح ميزة جديدة:

1. **اشرح الفائدة** - لماذا نحتاج هذه الميزة؟
2. **قدم مثالاً** - كيف ستُستخدم؟
3. **اذكر البدائل** - هل هناك حلول أخرى؟

## 🔄 خطوات المساهمة

### 1. قم بالتعديلات
```bash
# قم بتحريرات الملفات
# ثم اختبر محليًا
npm run dev
```

### 2. فحص الأخطاء
```bash
npm run lint
```

### 3. قم بـ Commit
```bash
git add .
git commit -m "feat: إضافة ميزة جديدة" 
# أو
git commit -m "fix: إصلاح خطأ في الشاشة الرئيسية"
```

### 4. اضغط البيانات
```bash
git push origin feature/اسم-الميزة
```

### 5. افتح Pull Request
على GitHub:
1. اذهب إلى المستودع
2. اختر "Pull Requests"
3. اضغط "New Pull Request"
4. اختر فرعك
5. أضف وصفاً واضحاً

---

## 📋 أنواع الرسائل (Commit Messages)

اتبع هذا الأسلوب:

```
feat: إضافة ميزة جديدة (Feature)
fix: إصلاح خطأ (Bug Fix)
docs: تحديثات التوثيق
style: تغييرات التنسيق والأسلوب
refactor: إعادة هيكلة الكود
test: إضافة أو تعديل الاختبارات
perf: تحسينات الأداء
ci: تغييرات في CI/CD
```

**أمثلة:**
```
feat: إضافة نظام الإشعارات
fix: إصلاح مشكلة تسجيل الدخول
docs: تحديث ملف README
```
