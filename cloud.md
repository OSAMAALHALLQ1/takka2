## 1. ملاحظات الجلسة — ما تم إنجازه

### الهدف المطلوب
إعادة تصميم شاشة التحميل (Loading Screen) لتكون أكثر احترافية وسرعة واتساقاً مع شعار الموقع "تكة"، مع فهم معنى الشعار.

### المشاكل التي كانت موجودة في الشاشة القديمة
| المشكلة | التأثير |
|---|---|
| `backdrop-filter: blur(20px)` | استهلاك عالٍ للأداء على الأجهزة الضعيفة |
| radial-gradient ضخم (600px) + `bgBreathe` لمدة 6 ثوانٍ | استهلاك غير ضروري للموارد |
| `dotBounce` (النقطة الحمراء تقفز) | غير منطقي — النقطة هي "زر" وليس كرة قابلة للقفز |
| 6 animations منفصلة غير متزامنة | تعقيد وصعوبة صيانة |
| marquee progress bar | عنصر عام لا يمت بصلة لهوية العلامة |

### الحل المطبّق
- **فهم الشعار**: الشعار يمثل **"تكة"** = علامة الضغط على زر (النقطة الحمراء هي الزر، والأشكال الداكنة هي الإصبع يضغط الزر).
- **الأنيميشن الجديد**: يحاكي ضغط الإصبع على الزر — الإصبع ينزل على الزر، الزر يتقلص مع توهّج أحمر، ثم يرتد ويعود.
- **الأداء**: حركات GPU فقط (`transform`, `opacity`, `filter`) — لا blur، لا gradients متحركة كبيرة.

## 3. تحليل الشعار

الشعار عبارة عن تكوين فني يمثل مفهوم "تكة" (الضغطة على الزر):

```
       ●   ← النقطة الحمراء = "الزر" (لون: #C1272D / #dc2626)
       │
    [أشكال]  ← الأشكال الداكنة = "الإصبع" يضغط الزر (لون: #262424 / #18181b)
```

**الملفات المرجعية للشعار** (في `Downloads/`):
- `شعار تكة اساسي.svg` — الشعار الأساسي الكامل
- `تكة نص.svg` / `تكة نص 2.svg` — النص "تكة"
- `ايقونة الموقع.svg` / `ايقونة الموقع2.svg` — أيقونة الموقع
- `takka logo.svg` — شعار كامل بالألوان
- `2224.svg` / `تكة 2/3/4.svg` — إصدارات إضافية

### ملفات الشعار في المشروع (`public/`)
| الملف | الاستخدام |
|---|---|
| `favicon.svg` | أيقونة تبويب المتصفح |
| `logo-emblem.svg` | رمز العلامة التجارية |
| `logo-full.svg` | الشعار الكامل بالنص |
| `icons.svg` | أيقونات إضافية |
| `manifest.webmanifest` | ملف PWA (تطبيق قابل للتثبيت) |
| `sw.js` | Service Worker (تخزين مؤقت يعمل دون اتصال) |

## 7. قاعدة البيانات (Supabase)

التكامل مع Supabase **عميق وشامل** على ثلاث طبقات:

### (أ) إعداد العميل
- **الملف:** `src/utils/supabaseClient.js`
- يقرأ `VITE_SUPABASE_URL` و `VITE_SUPABASE_ANON_KEY` من متغيرات البيئة.
- يعيد `null` إذا لم يُضبط، فيعمل التطبيق في وضع **LocalStorage فقط (Offline-first)**.

### (ب) الجداول (7 جداول في `supabase_schema.sql`)
| الجدول | الوظيفة |
|---|---|
| `tables` | خريطة الصالة (70 طاولة ثابتة، سعة 4/6/8، مناطق: داخلي/خارجي/تراس) |
| `employees` | الموظفون: أدوار، رموز، كلمات مرور، رواتب |
| `menu` | أصناف القائمة: أسماء عربي/إنجليزي، أسعار، أقسام |
| `departments` | الأقسام: مطبخ، بار، شيشة |
| `bills` | الفواتير المغلقة: عناصر، إجماليات، طرق دفع، طوابع زمنية |
| `notifications` | إشعارات موجّهة حسب الدور |
| `dept_orders` | الطلبات النشطة للأقسام |

### (ج) دوال PostgreSQL الذرية (RPCs)
- `submit_order_atomic()` — إنشاء طلب بشكل idempotent، تحديث حالة الطاولة، قفل صفوف `FOR UPDATE`.
- `close_table_invoice_atomic()` — إنشاء فاتورة، إغلاق الطاولة، حذف الطلبات النشطة.

### (د) الميزات المتقدمة
- **Realtime**: كل الجداول في publication للبث اللحظي → تزامن متعدد الأجهزة.
- **Session Heartbeat**: كل 5 ثوانٍ يفحص Supabase؛ إذا سجّل موظف دخوله على جهاز آخر → يُطرد (منع الجلسات المكررة).
- **SQL Migrations** في `src/db/`: `supabase_saas_migration.sql`, `production_hardening.sql`, `archive_migration.sql`.

## 10. تفاصيل شاشة التحميل الجديدة

### الفلسفة
الشعار = "إصبع يضغط زراً أحمر". الأنيميشن يحاكي هذا الإجراء — ضغطة لطيفة على الزر.

### البنية (HTML)
```html
<div class="initial-loader">
  <div class="loader-card">
    <svg class="loader-logo">
      <circle class="logo-dot"/>     ← الزر الأحمر (يُضغط)
      <g class="logo-hand">          ← الإصبع (ينزل ويضغط)
        <path class="logo-path"/>
        <path class="logo-path"/>
      </g>
      <g class="logo-body">          ← جسم الشعار (ثابت)
        <path class="logo-path"/>
        <path class="logo-path"/>
      </g>
    </svg>
    <div class="loader-text">تكة</div>
    <div class="loader-sub">نظام إدارة المطعم</div>
  </div>
</div>
```

### حركات الأنيميشن (CSS Keyframes)

**1. `tk-press` — ضغط الزر (دورة 1.8 ثانية):**
| النسبة | الحالة |
|---|---|
| 0%, 100% | `scale(1)` — وضع الراحة |
| 30%–55% | `scale(0.78)` + توهّج أحمر — لحظة الضغط |
| 70% | `scale(1.08)` — ارتداد خفيف |
| 85% | يعود للحجم الطبيعي |

**2. `tk-hand` — حركة الإصبع (دورة 1.8 ثانية):**
| النسبة | الحالة |
|---|---|
| 0%, 100% | `translateY(0)` — الإصبع فوق |
| 25%–55% | `translateY(18px)` — الإصبع ينزل ويضغط |

**3. `tk-once` — الظهور الأول (مرة واحدة):**
العناصر تظهر بالتدريج بـ opacity ثم تبقى ثابتة (لا تومضض).

### التحسينات
- ✅ **حركات GPU فقط** (`transform`, `opacity`, `filter`) — لا reflow.
- ✅ **لا `backdrop-filter`** — أسرع بكثير.
- ✅ **لا gradients ضخمة** — توفير في الذاكرة.
- ✅ **`will-change`** على العناصر المتحركة.
- ✅ **`prefers-reduced-motion`** — يحترم تفضيل المستخدم ويعطّل الحركة.
- ✅ **النص ثابت** — لا يومض (مشكلة الشاشة القديمة حُلّت).

### مكان التطبيق
- **`index.html`**: شاشة التحميل الأولى (قبل تحميل React) + الـ CSS.
- **`src/App.jsx`** (الأسطر 595–615): شاشة التحميل الثانية (أثناء تهيئة قاعدة البيانات).


### إعداد Supabase (اختياري)

```env

```
ثم نفّذ `supabase_schema.sql` في محرر SQL الخاص بـ Supabase. بدونه، يعمل التطبيق بـ LocalStorage فقط.
