## 2. تحليل الشعار

الشعار عبارة عن تكوين فني يمثل مفهوم **"تكة"** (الضغطة على الزر):

```
       ●   ← النقطة الحمراء = "الزر" (لون: #C1272D / #dc2626)
       │
    [أشكال]  ← الأشكال الداكنة = "الإصبع" يضغط الزر (لون: #262424 / #18181b)
```

### بنية ملف SVG
- **`viewBox="0 0 1085.6 1146.4"`** — إحداثيات العرض.
- **دائرة الزر** `<circle cx="268.6" cy="202.1" r="86.9">` — مكتوبة **مباشرةً** في فضاء الـ viewBox (بلا تحويل).
- **مسارات اليد والبطاقة البيضاء** — مكتوبة في **فضاء إحداثي مزاح** (مثل `M1903,1996.8`)، وكل مسار يحمل `transform="translate(-987.9 -850.4)"` ليجلبه داخل الـ viewBox.

> ⚠️ هذا التمايز بين فضاءَي الإحداثيات (الزر داخل viewBox، اليد والبطاقة مزاحتان) هو جوهر الإصلاح الموضّح في §10. أي تحريك يجب أن يحترم هذا التمايز وإلا يختفي الشعار أو يبدو عملاقاً.

### ملفات الشعار المرجعية (في `Downloads/`)
- `ايقونة الموقع2.svg` — أيقونة الموقع المستخدمة في شاشة التحميل (المرجع الأساسي).
- `شعار تكة اساسي.svg`, `تكة نص.svg`, `takka logo.svg`, `2224.svg` — إصدارات إضافية.

### ملفات الشعار في المشروع (`public/`)
| الملف | الاستخدام |
|---|---|
| `favicon.svg` | أيقونة تبويب المتصفح |
| `logo-emblem.svg` | رمز العلامة التجارية |
| `logo-full.svg` | الشعار الكامل بالنص |
| `manifest.webmanifest` | ملف PWA (تطبيق قابل للتثبيت) |
| `sw.js` | Service Worker (تخزين مؤقت يعمل دون اتصال) |

## 6. قاعدة البيانات (Supabase)

التكامل مع Supabase **عميق وشامل** على ثلاث طبقات:

### (أ) إعداد العميل
- **الملف:** `src/utils/supabaseClient.js`
- يقرأ `VITE_SUPABASE_URL` و `VITE_SUPABASE_ANON_KEY` من متغيرات البيئة.
- يعيد `null` إذا لم تُضبط، فيعمل التطبيق في وضع **LocalStorage فقط (Offline-first)**.

### (ب) الجداول (في `supabase_schema.sql`)
| الجدول | الوظيفة |
|---|---|
| `tables` | خريطة الصالة (طاولات ثابتة، سعة 4/6/8، مناطق: داخلي/خارجي/تراس) |
| `employees` | الموظفون: أدوار، رموز، كلمات مرور، رواتب |
| `menu` | أصناف القائمة: أسماء عربي/إنجليزي، أسعار، أقسام |
| `departments` | الأقسام: مطبخ، بار، شيشة |
| `bills` | الفواتير المغلقة: عناصر، إجماليات، طرق دفع، طوابع زمنية |
| `notifications` | إشعارات موجّهة حسب الدور |
| `dept_orders` | الطلبات النشطة للأقسام |

### (ج) دوال PostgreSQL الذرية (RPCs)
- `submit_order_atomic()` — إنشاء طلب بشكل idempotent، تحديث حالة الطاولة، قفل صفوف `FOR UPDATE`.
- `close_table_invoice_atomic()` — إنشاء فاتورة، إغلاق الطاولة، حذف الطلبات النشطة.

### (د) الميزات المتقدمة
- **Realtime**: الجداول في publication للبث اللحظي → تزامن متعدد الأجهزة.
- **Session Heartbeat**: كل 5 ثوانٍ يفحص Supabase؛ إذا سجّل موظف دخوله على جهاز آخر → يُطرد (منع الجلسات المكررة).
- **SQL Migrations** في `src/db/`: `supabase_saas_migration.sql`, `production_hardening.sql`, `archive_migration.sql`.

## 9. شاشتا التحميل (Loading Screens)

هناك **شاشتا تحميل متتابعتان** لضمان عدم وميض شاشة بيضاء:

```
[فتح الموقع] → Loader #1 (CSS) ──> React يُحمّل ──> Loader #2 (motion) ──> التطبيق
                index.html          main.jsx         App.jsx (DB init)
```

### 🔹 Loader #1 — CSS (قبل تحميل React)
- **الموقع:** `index.html` (مضمن داخل `<style>` و `<div id="root">`).
- **الغرض:** يظهر فوراً قبل أن يُحمَّل ويُوزن كود React.
- **التقنية:** CSS Keyframes خالصة — لا JavaScript.
- **الحركة:**
  - `tk-press` — الزر يتقلص (`scale(0.78)`) مع توهّج أحمر ثم يرتد.
  - `tk-hand` — الإصبع ينزل (`translateY(18px)`) ويضغط الزر.
  - `tk-once` — عناصر الشعار والنص تظهر بـ opacity مرة واحدة ثم تبقى ثابتة.
- **الدورة:** ~1.8 ثانية، تكرار لا نهائي.
- **`prefers-reduced-motion`**: تُعطَّل كل الحركات.
- **الترتيب البصري للـ SVG:** `logo-dot` (الزر) + `logo-hand` (الإصبع: مساران) + `logo-body` (جسم الشعار ثابت).

### 🔹 Loader #2 — React + motion (أثناء تهيئة قاعدة البيانات)
- **الملف:** `src/components/Common/LogoLoader.jsx`.
- **الاستدعاء:** `App.jsx` — يُعرض طالما `!databaseReady` (انظر `if (!databaseReady) return <LogoLoader />;`).
- **التقنية:** `motion/react` (Framer Motion) — تحريك `transform` فقط (GPU، بلا reflow).
- **التكوين:**
  - `<g className="logo-background">` — البطاقة البيضاء (ثابتة).
  - `<motion.circle className="red-button">` — الزر الأحمر، نَبضة `scale: [1,1,1,0.94,1]`.
  - `<motion.g className="hand-group">` — **كل اليد** كوحدة واحدة تصعد ثم تضغط.
- **الحركة (دورة 2.2 ثانية، `ease: [0.22, 1, 0.36, 1]`):**

| المسار/العنصر | keyframes |
|---|---|
| اليد — `y` | `[260, 120, 28, -8, 0]` (تصعد من الأسفل، ضغطة `-8` في النهاية) |
| اليد — `x` | `[20, 14, 5, 0, 0]` |
| اليد — `rotate` | `[4, 2, 1, -1, 0]` (ميلان خفيف يزول) |
| الزر — `scale` | `[1, 1, 1, 0.94, 1]` (نَبضة عند اللمس) |

- **المؤقتات (`times`):** اليد `[0, 0.45, 0.82, 0.92, 1]`، الزر `[0, 0.82, 0.88, 0.94, 1]`.
- **القِياس والتمركز:** بطاقة ثابتة **220px**، `position: fixed; inset: 0`، flex-centered، خلفية متدرجة ناعمة، ظل ناعم، خط Cairo.
- **`prefers-reduced-motion`**: تتحول إلى حلقة تلاشٍ هادئة (opacity `[0.4, 1, 0.4]`) بلا إزاحة.
- **إتاحة الوصول:** `role="img"` + `aria-label` عربي على الشعار، `aria-hidden` على الـ SVG.

> 📐 تفصيل هندسي مهم: في Loader #2 يُفصل بين «فضاء الإحداثيات» و«فضاء التحريك» — `<g>` داخلي يحمل `translate(-987.9 -850.4)` (لإدخال مسارات اليد/البطاقة إلى الـ viewBox)، بينما `motion.g` الخارجي يحمل *فقط* تحريك الأنيميشن. الزر الأحمر يبقى خارج `<g>` المزاح لأنه أصلاً داخل الـ viewBox. هذا الفصل هو ما يمنع تعارض التحويلين (انظر §10).

## 10. سجل إصلاحات شاشة التحميل

### المشاكل التي تم إصلاحها (مرجعية)

| # | المشكلة | السبب الجذري | الإصلاح |
|---|---|---|---|
| 1 | **اليد مفقودة والشعار لا يكتمل** | مسارات اليد مكتوبة في فضاء إحداثي مزاح (`M1355…`, `M1881…`)؛ كانت تُرسم خارج الـ viewBox فيختفي الشعار كله ولا تظهر إلا الدائرة الحمراء. | إعادة تطبيق `transform="translate(-987.9 -850.4)"` عبر `<g>` داخلي لكل مجموعة مسارات مزاحة، مع إبقاء الزر خارجها. |
| 2 | **الشعار ضخم/يملأ الشاشة** | اختفاء البطاقة البيضاء (نفس سبب #1) جعل الدائرة منفردة وكبيرة. | نتيجة طبيعية لإصلاح #1 — عاد الشعار الكامل بأبعاده الصحيحة. |
| 3 | **يد ناقصة الأصابع** | نسخة سابقة حذفت مسار راحة اليد الكبير (`M1881…`). | إعادة جميع مسارات اليد الأربعة (إبهام + سبابة + راحة + مفصِل). |
| 4 | **`transformOrigin` هش للزر** | استخدام `268.6px 202.1px` قد يخطئ التوسيط في بعض المتصفحات مع تحويلات SVG. | `transformBox: 'fill-box'` + `transformOrigin: 'center'` للتوسيط المضمون. |
| 5 | **خطأ ESLint: `React` غير مستخدم** | الاستيراد القديم غير ضروري مع تحويل JSX الجديد. | حذف الاستيراد. |
| 6 | **خطأ ESLint: `setState` داخل `useEffect`** | قراءة `matchMedia` بشكل متزامن داخل التأثير. | نقل القراءة الأولى إلى مُهيِّئ كسول `useState(() => …)`، ويكتفي التأثير بالاشتراك. |

### مبادئ الحركة الملتزَم بها
- ✅ **حركات GPU فقط** (`transform`, `opacity`, `filter`) — لا reflow.
- ✅ **لا `backdrop-filter` ثقيل** ولا gradients متحركة ضخمة.
- ✅ **`will-change`** على العناصر المتحركة.
- ✅ **`prefers-reduced-motion`** يُحترم بالكامل.
- ✅ **النص ثابت** بعد الظهور (لا يومضض).
- ✅ **اليد كتلة واحدة** لا مسارات منفصلة.

### التحقق
- ✅ ESLint نظيف (0 أخطاء).
- ✅ البناء للإنتاج ناجح (`npm run build`).
